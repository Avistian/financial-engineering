Returns & the Stylized Facts

Lesson 004 · Foundations · ~40 min · has lab

Prices are non-stationary and awkward to model; returns are the currency of quant research. But returns are not the tame Gaussian variables intro stats assumes — they obey a set of stubborn empirical regularities called the stylized facts (Cont 2001). Every model you build must respect them, and half of all beginner mistakes come from ignoring them.

This lesson's single skill: compute log returns and identify the stylized facts in real data — fat tails, absent linear autocorrelation, and volatility clustering.

Simple vs log returns

R_t = (P_t − P_{t−1}) / P_{t−1}   ·   r_t = ln(P_t / P_{t−1}) = ln(1 + R_t)

Log returns add across time (a k-period return is the sum of the one-period log returns), are symmetric around zero, and play nicely with the Brownian-motion models of Q2. Simple returns add across assets (portfolio return = weighted sum). Quants default to log returns for time-series modeling. For small moves, r_t ≈ R_t.

The stylized facts you must respect

FactObservationConsequence
Fat tailsexcess kurtosis ≫ 0; crashes far more likely than Gaussian saysnormal-based risk models undercount tail loss
No linear autocorr.corr(r_t, r_{t+k}) ≈ 0naïve momentum on raw returns fails
Volatility clusteringcorr(\|r_t\|, \|r_{t+k}\|) > 0, decays slowlyvol is forecastable → GARCH, HAR-RV
Leverage effectdown moves raise future vol more than up movesasymmetric vol models (Q3)

The tension here is the entire game: direction is nearly unpredictable from past returns alone (no linear autocorrelation), yet volatility is quite predictable (clustering). Much of quant research is finding the nonlinear/cross-sectional structure that raw autocorrelation misses, while modeling volatility separately.

How to see them (the lab does this)

import numpy as np
r = np.log(px).diff().dropna()
r.kurtosis()            # >> 0   → fat tails
r.autocorr(lag=1)       # ~ 0    → no linear predictability
r.abs().autocorr(lag=1) # > 0    → volatility clustering
Trap. Reporting Sharpe ratios and t-statistics as if returns were IID normal. Fat tails and volatility clustering violate both assumptions, inflating apparent significance and making overfit strategies look "statistically real." This is the first leak through which false edges enter — and the reason the deflated Sharpe ratio (Year 2 Q2) exists. Distrust any single-number performance claim that ignores the stylized facts.
