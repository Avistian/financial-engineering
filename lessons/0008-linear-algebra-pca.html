Linear Algebra for Quants: Covariance, Eigendecomposition, PCA

Lesson 008 · Foundations · ~60 min

Warm up

Lessons 004–007 lived in one dimension at a time: the returns of a strategy, the Sharpe of a series, the p-value of a test. Real quant research is many assets at once — a book of hundreds of stocks, a grid of signals, a panel of returns. The moment you have more than one series, the object that governs risk, diversification, and structure is not a number but a matrix: the covariance matrix. This lesson gives you the linear algebra to read that matrix — not as a recipe to memorize, but as a geometry you can see.

This lesson's single skill: take a returns panel, form its covariance matrix, and decompose it into principal components — eigenvectors and eigenvalues — and say in plain English what each one means. That is Principal Component Analysis (PCA), and it is the workhorse behind the market factor, risk models, covariance denoising (Ledoit-Wolf / random matrix theory, Year 2–3), OFI cross-impact (unit 063), and portfolio construction (units 081–087). The payoff is the lab: you will run PCA on a real returns panel by hand (from the covariance matrix up), confirm that the first component is essentially "the market," and measure how much of the panel's risk lives in a handful of directions.

The returns panel is a matrix

Start concretely. Take d assets and record their (log) returns over n days. Stack them into an n × d matrix X: one row per day, one column per asset.

Xij = return of asset j on day i   (i = 1…n days, j = 1…d assets)

Every downstream object is built from X. A portfolio is a weight vector w (one weight per asset, d numbers); the portfolio's daily return series is the matrix–vector product Xw. The portfolio's variance — the thing risk management actually cares about — is wᵀΣw, where Σ (capital sigma) is the covariance matrix we build next. Linear algebra is not decoration here; it is the language in which "how risky is this book?" is asked and answered.

One convention first, because everything depends on it. We almost always center each column — subtract that asset's mean return so each column has mean zero. Centered data is written with a tilde, X̃. Centering matters because covariance and PCA are about how assets vary around their means and co-move; leaving the means in would let a large average return masquerade as "variation." (Later we also often scale each column to unit variance — more on that in the trap.)

The covariance matrix Σ

The covariance of two assets measures how they move together. For centered return columns ã and b̃ (each of length n):

Cov(A, B) = (1 / (n − 1)) · Σi ãi · b̃i

Read it as an average product of co-deviations: on days both assets are above their mean, the product is positive; when one is up and the other down, it is negative. If the positives dominate, the assets tend to move together (positive covariance); if the negatives dominate, they move oppositely. The n − 1 (not n) is Bessel's correction — the same unbiased-variance denominator you met in Lesson 006, because we estimated each mean from the same data. Note Cov(A, A) = Var(A): an asset's covariance with itself is just its variance.

The covariance matrix Σ collects every pairwise covariance into a d × d grid: entry Σjk = Cov(asset j, asset k). In compact matrix form, for centered data:

Σ = (1 / (n − 1)) · X̃ᵀ X̃   (a d × d symmetric matrix)

Two structural facts about Σ that we lean on for the rest of the lesson: it is symmetric (Cov(A,B) = Cov(B,A), so Σ = Σᵀ), and it is positive semi-definite (wᵀΣw ≥ 0 for any w — a variance can never be negative). Those two properties are exactly what make its eigendecomposition so clean, as we will see.

Worked example (two assets, five days). Centered returns (in %): ã = (+1, −2, +2, −1, 0), b̃ = (+1, −2, +1, −1, +1). Then Var(A) = (1+4+4+1+0)/4 = 2.5, Var(B) = (1+4+1+1+1)/4 = 2.0, and Cov(A,B) = (1·1 + (−2)(−2) + 2·1 + (−1)(−1) + 0·1)/4 = 8/4 = 2.0. So
Σ = [ [2.5, 2.0], [2.0, 2.0] ]
The correlation (covariance rescaled to [−1, 1]) is ρ = Cov / (σAσB) = 2.0 / √(2.5·2.0) = 2.0/√5 ≈ 0.89 — these two assets are strongly co-moving. Hold onto this Σ; we decompose it by hand below.

Covariance vs correlation — and why it bites. Covariance carries units (return²) and scale: a high-volatility asset has large covariances with everything simply because it is loud, not because it is specially related. Correlation divides that out (Corr = Σjk / (σjσk)), so it is comparable across pairs. This distinction is not academic: PCA run on the raw covariance matrix will be dominated by whichever assets happen to be most volatile. We return to this in the trap — it is the single most common way to run PCA wrong.

Eigenvectors and eigenvalues: the special directions of a matrix

A matrix M is a transformation: feed it a vector v and it returns another vector Mv, generally pointing somewhere new. For most directions the matrix rotates and stretches. But some special directions are only stretched, not rotated. A nonzero vector v is an eigenvector of M if

M v = λ v

where the scalar λ (lambda) is the eigenvalue — the factor by which that direction is stretched. "Eigen" is German for "own" or "characteristic": these are the matrix's own, characteristic directions. Geometrically: apply M to an eigenvector and it lands on the same line it started on, just λ times as long (flipped if λ < 0). Every other vector gets knocked off its line.

For a symmetric matrix — and a covariance matrix is always symmetric — the spectral theorem guarantees three beautiful things, and they are the entire reason PCA works:

  • All eigenvalues are real (no imaginary numbers to interpret).
  • Eigenvectors of distinct eigenvalues are orthogonal — mutually perpendicular, at right angles. They form a clean set of axes.
  • A d × d symmetric matrix has a full set of d orthogonal eigenvectors that span the whole space: any vector can be written as a combination of them. This is the eigendecomposition Σ = QΛQᵀ, where the columns of Q are the orthonormal eigenvectors and Λ is the diagonal matrix of eigenvalues.

Because Σ is also positive semi-definite, all its eigenvalues satisfy λ ≥ 0. That is what lets us read each eigenvalue as a variance — a quantity that cannot be negative.

Worked example (decomposing our Σ by hand). For a 2 × 2 symmetric Σ = [[a, b], [b, c]] the eigenvalues solve the quadratic λ = (tr/2) ± √((tr/2)² − det), with trace tr = a + c and determinant det = ac − b². For Σ = [[2.5, 2.0], [2.0, 2.0]]: tr = 4.5, det = 2.5·2.0 − 2.0² = 5 − 4 = 1, so λ = 2.25 ± √(5.0625 − 1) = 2.25 ± 2.016, giving λ₁ ≈ 4.27 and λ₂ ≈ 0.23. The first eigenvector (solving (Σ − λ₁I)v = 0) comes out to v₁ ≈ (0.75, 0.66) — both components positive, i.e. the "both assets rise together" direction. The second, v₂ ≈ (−0.66, 0.75), is perpendicular to it — the "A up, B down" spread direction. Sanity check: λ₁ + λ₂ = 4.27 + 0.23 = 4.5 = tr(Σ) = Var(A) + Var(B). The trace is conserved.

The covariance ellipse — seeing the decomposition

Here is the picture that makes all of this click. A cloud of two correlated returns forms a tilted elliptical blob. That ellipse is the covariance matrix drawn out: its axes point along the eigenvectors of Σ, and the length of each axis is √λ — the standard deviation of the data along that direction. The longest axis is the first principal component, the direction of maximum variance. Drag the correlation and watch the ellipse — and its principal axes — respond:

The teal cloud is two assets' returns. The gold curve is the 2σ covariance ellipse. The red arrow is PC1 (the eigenvector with the larger eigenvalue — longest axis, most variance); the teal arrow is PC2, exactly perpendicular. At ρ = 0 the ellipse is upright and the PCs line up with the raw axes; as |ρ| → 1 it collapses toward a line and PC1 swallows almost all the variance — that is the redundancy PCA is built to find.

Notice what the two eigenvalues are telling you: λ₁ is the variance if you looked at the data along the long axis, λ₂ the variance along the short axis. PCA is nothing more than "rotate your coordinate system so the axes line up with the ellipse." In the new coordinates the assets are uncorrelated (the ellipse is axis-aligned) and sorted by how much they vary. No new information is created; it is re-expressed along its natural axes.

PCA = the eigendecomposition of Σ

We can now state Principal Component Analysis exactly, with no hand-waving:

  1. Center the returns panel (and usually standardize — see the trap).
  2. Form the covariance (or correlation) matrix Σ.
  3. Compute its eigenvectors and eigenvalues, sorted so λ₁ ≥ λ₂ ≥ … ≥ λd.
  4. The eigenvectors are the principal components (PCs) — the directions; the eigenvalues are the variance captured along each.

Each principal component is a fixed portfolio of the original assets (its eigenvector is a set of weights), and these portfolios are mutually uncorrelated and ordered from most to least variance. The fraction of the panel's total risk captured by PC k is its eigenvalue over the sum of all eigenvalues:

variance explained by PCk = λk / (λ₁ + λ₂ + … + λd) = λk / tr(Σ)

The denominator is the trace of Σ — the sum of the diagonal, i.e. the total variance of all assets — and it equals the sum of the eigenvalues (as our worked example confirmed, 4.27 + 0.23 = 4.5). So in that two-asset example, PC1 explains 4.27 / 4.5 ≈ 95% of all variation: two correlated assets are, to 95%, one underlying bet plus a little noise. That collapse from "two numbers" to "essentially one direction" is the entire point — and it only gets more dramatic as the number of assets grows.

The scree plot and the market factor

Plot the sorted eigenvalues as bars and you get a scree plot (named for the rubble at the foot of a cliff — a steep drop then a flat tail). For a real equity panel the picture is striking: one giant bar, then a long low tail. Below is the exact spectrum of an idealized panel where every pair of assets shares the same correlation ρ — the closed-form case λ₁ = 1 + (d−1)ρ, λ₂ = … = λd = 1 − ρ. Raise ρ and watch PC1 devour the panel:

Bars are eigenvalues (the scree); the red bar is PC1. The gold line is cumulative variance explained. At ρ = 0 every bar is equal height (a flat scree — no structure, nothing to compress). As ρ rises, PC1 alone explains (1 + (d−1)ρ)/d of everything: with 20 assets at ρ = 0.4 that is already ~43%, and at ρ = 0.7 it is ~71%.

That dominant first component has a name in markets: it is the market factor. Its eigenvector loads with the same sign on essentially every stock — it is the "everything moves together" direction, the tide that lifts or sinks all boats. Empirically, on daily equity returns PC1 explains anywhere from ~25% (calm, diversified panels) to well over 50% (crises, when correlations spike toward 1 and the whole market becomes a single trade). This is why "diversification fails exactly when you need it" — a stress event is ρ → 1, which is λ₁ → d: the panel degenerates to one factor and there is nowhere to hide.

The components after the first are interpretable too. PC2 on a stock panel is often a sector or style spread (long one group, short another — it loads with opposite signs); later PCs pick up finer structure, and the long flat tail is mostly noise. Deciding where signal ends and noise begins is the subject of random matrix theory (Year 2–3, units 082–083) — previewed in the trap.

Projection and reconstruction: PCA as compression

Once you have the principal directions, you can project the data onto them. The projection of the return panel onto the PCs gives the scores (also called factor returns): Z = X̃ Q, where column k of Z is the daily return of the k-th principal-component portfolio. PC1's score series is, in the market example, essentially a market-index return built purely from the data's own covariance structure.

The power move is dimensionality reduction: keep only the first k components (the ones with meaningful eigenvalues) and discard the noisy tail. You can then reconstruct an approximation of the original panel from just those k directions, X̃ ≈ Z1:k Q1:kᵀ. The Eckart–Young theorem guarantees this is the best possible rank-k approximation in a least-squares sense — no other k-factor model reproduces the panel with smaller error. That is why a handful of PCs can stand in for hundreds of assets in a risk model: you are keeping the directions that carry the variance and throwing away the rubble.

Explain it back

Trap — the four ways PCA lies to you. (1) Scale. Run PCA on the raw covariance of assets with wildly different volatilities and PC1 will just be "the loudest asset," not a real common factor. Fix: standardize each column to unit variance first (equivalently, decompose the correlation matrix). Forgetting this is the #1 PCA mistake. (2) Non-stationarity. Σ is not a constant of nature — correlations drift and, in crises, spike toward 1. A PCA fit on last year's calm data mis-states today's risk; the market factor you estimated in June is not the one that hits you in a September sell-off. Always ask "over what window was this covariance estimated, and is it still valid?" (3) Noise eigenvalues. With d assets and only n days, most small eigenvalues are pure estimation noise, not real directions — especially when d is comparable to n. Random matrix theory (Marchenko–Pastur) predicts the noise band; eigenvalues inside it carry no signal. Trading the tiny-eigenvalue "components" means trading noise, and inverting a noisy Σ (for portfolio optimization) blows up on exactly those directions — the motivation for Ledoit–Wolf shrinkage and RMT denoising in Year 2–3. (4) Leakage. Fitting PCA on the entire sample and then using those components in a backtest is look-ahead bias: your "factors" were built with knowledge of the future. In a real backtest the covariance must be estimated only from data available up to each point in time. This is the same leakage discipline as Lesson 007's multiple-testing warning, wearing a matrix hat.

Common questions (and honest answers)

“Are principal components the same thing as the real economic factors, like value or momentum?”
No. PCs are statistical factors — the directions of maximum variance in this dataset, defined purely by the covariance matrix. They are unsupervised and have no idea what "value" means. PC1 usually coincides with the market because the market really is the dominant source of common variance, but PC2, PC3… rarely line up cleanly with named style factors. Economic factor models (Fama–French, etc.) are chosen a priori; PCA factors are discovered from the data. Both are useful; don't confuse them.
“Why decompose Σ instead of just running SVD on the data matrix X?”
They are the same thing. The singular value decomposition of the centered data X̃ gives right singular vectors that are exactly the eigenvectors of X̃ᵀX̃ ∝ Σ, and the singular values squared are the eigenvalues. SVD is the numerically preferred route (it never forms Σ explicitly, so it is more stable); eigendecomposition of Σ is the conceptually clearest route and what you'll do by hand in the lab. Same answer, two doors.
“The eigenvector's sign flipped when I re-ran it — is my code wrong?”
No. An eigenvector is only defined up to sign: if v satisfies Σv = λv then so does −v. Solvers pick a sign arbitrarily. It never changes the variance explained or the subspace; it only flips the interpretation of "long vs short" on that component. Fix a convention (e.g. make the largest loading positive) if you need reproducibility.
“How many components should I keep?”
There's no single rule, and anyone who gives you one is overselling. Common heuristics: keep components until cumulative variance hits some target (say 90%); look for the "elbow" in the scree plot; or — the principled one for finance — keep only eigenvalues above the random-matrix-theory noise band, since the rest are statistically indistinguishable from noise. The honest answer is "it depends on why you're reducing," and the RMT route (Year 2–3) is the defensible one.
“If PC1 is basically the market, can't I just use an index and skip PCA?”
For hedging the market, often yes. But PCA earns its keep beyond PC1: measuring how concentrated risk is (is 80% of my book one bet?), building uncorrelated factor portfolios, cleaning the covariance matrix before optimization, and detecting regime shifts (when PC1's share suddenly jumps, correlations are spiking). The index is one output; PCA is the machine that also tells you everything the index leaves out.
